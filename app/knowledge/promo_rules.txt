# 电商促销规则手册（测试知识库示例）

## 1. 满减活动规则

### 1.1 基本规则
- 满减以订单实付金额（扣除其他优惠后）计算，不含运费
- 单笔订单最多叠加一档满减，不支持多档累加
- 示例：满100减10、满200减30，同一订单只能享受一档

### 1.2 金额边界
- 正好满足门槛：金额等于门槛时可享受优惠（含等于，即 >= 门槛）
- 差1分不满足：实付199.99元不触发满200减30
- 凑单场景：多件商品合计需达到门槛，单件不够可添加其他商品凑单

### 1.3 叠加规则
- 满减与平台优惠券：不可叠加（二选一，默认选优惠力度更大的）
- 满减与店铺优惠券：可叠加（先减满减，再减优惠券）
- 满减与会员折扣：不可叠加（二选一）
- 满减与运费险：不影响（运费险独立计算）

### 1.4 历史典型缺陷
- [BUG-2024-0312] 满减金额上限未生效：单笔可减超过上限金额
  根因：promotionService 未正确读取 maxDiscountAmount 字段
  修复版本：v2.3.1
- [BUG-2024-0601] 凑单后取消部分商品，满减未重新计算
  根因：订单变更事件未触发促销引擎重算
  修复版本：v2.4.0

## 2. 优惠券规则

### 2.1 券类型
- 平台通用券：全品类可用
- 店铺专属券：仅限指定店铺使用
- 品类券：仅限指定品类商品

### 2.2 使用限制
- 每个用户每张券限用1次
- 活动期间每用户最多使用3张同类券
- 过期券：到期时间精确到秒，服务端以请求到达时间为准

### 2.3 边界场景（高频测试点）
- 券的最后1秒使用：23:59:59 发起支付，00:00:00 到达服务端 → 应提示券已过期
- 并发使用同一张券：两个设备同时下单使用同一张券 → 仅允许一次成功（幂等）
- 退款后券的处理：使用优惠券的订单退款后，券应自动退还

## 3. 幂等设计规范

### 3.1 核心原则
同一业务操作，无论执行多少次，结果只产生一次。

### 3.2 电商核心场景的幂等要求
- 下单接口：相同 clientOrderId 的重复请求，只创建一笔订单
- 支付接口：相同 paymentRequestId 的重复请求，只扣款一次
- 优惠券核销：相同 userId + couponId + orderId，只核销一次
- 库存扣减：基于订单 ID 的幂等键，防止超扣

### 3.3 常见测试点
- 网络超时重试：客户端超时后重试，服务端已处理的不重复处理
- 消息队列重复消费：MQ 消息重复投递，消费端幂等处理
- 用户快速双击：前端防抖 + 后端幂等双重保障

---

## 4. 库存规则

### 4.1 超卖防护
- 预占库存：下单时预占，支付超时释放，支付成功扣减
- 超卖阈值：库存为0时不允许下单（返回"库存不足"）
- 大促场景：启用 Redis 原子扣减，防止 DB 并发超卖

### 4.2 边界场景
- 最后1件：并发请求只允许1次成功，其余返回库存不足
- 库存从0恢复（补货）：触发缺货订阅用户通知
